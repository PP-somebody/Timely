# 地震历史事件时间轴 - 需求拆解文档

## 产品概述

- **产品类型**: 信息可视化展示页
- **场景类型**: <scene_type>info_viz</scene_type>
- **目标用户**: 地震研究者、科普爱好者、公众
- **核心价值**: 以可交互时间轴形式直观展示地震事件全过程，支持 JSON 数据驱动
- **界面语言**: 中文
- **主题偏好**: 浅色
- **导航模式**: 无导航（单页文档流布局）

---

## 内容理解

### 核心主题
汶川地震全过程时间轴，按时间升序展示地震发生、应急响应等关键事件节点。

### 章节结构
1. **页面标题区** - 展示时间轴主题（从 JSON `topic` 字段读取）
2. **时间轴主体** - 按时间升序排列的事件节点，每个节点含时间、标题、摘要、阶段标签
3. **事件详情展开区** - 点击节点后展示详细说明、地点、来源链接

### 重点内容
- **核心结论**: 汶川地震从发生到应急响应的关键时间节点
- **关键决策**: 每个事件的阶段标签（phase）帮助理解事件所处阶段

---

## 可视化识别

### 需要可视化的内容

| 内容位置 | 信息关系 | 可视化方式 | 理由 |
|---------|---------|-----------|-----|
| 全部事件 | 时间序列 | 时间轴（竖线 + 圆点节点 + 卡片） | 事件按时间先后排列，≥3 个时间节点，符合时间轴使用条件 |

### 保留文字的内容
- 事件详细说明（detail）保留文字段落形式，在展开卡片中展示
- 来源链接保留超链接形式

---

## 数据与图表规划

### 数据概览
- **数据结构**: JSON 对象，含 `topic`（字符串）和 `events`（数组）
- **事件字段**: `time`、`title`、`summary`、`detail`、`phase`、`location`、`importance`、`sources`
- **数据量**: 初始 mock 数据含 2 个事件，后续可扩展至任意数量

### 图表规划
本需求为纯时间轴展示，不涉及统计图表。时间轴本身即为核心可视化形式。

## 功能列表

- **页面整体**:
  - **页面目标**: 以可交互时间轴形式展示地震事件全过程，支持展开/收起查看详情
  - **功能点**:
    - **数据驱动渲染**: 页面打开后自动调用 `loadTimelineData()` 加载 JSON 数据，根据数据动态生成时间轴节点，不硬编码事件内容
    - **时间轴可视化**: 以竖线时间轴 + 圆点节点 + 卡片式内容展示事件，按 `time` 字段升序排列
    - **节点信息展示**: 每个节点卡片默认显示时间、标题、摘要、阶段标签（phase badge）
    - **展开/收起详情**: 点击节点卡片后展开，显示详细说明（detail）、地点（location）、来源链接列表（sources）
    - **来源链接跳转**: 每个来源链接可点击，`target="_blank"` 新窗口打开新闻报道或官方资料
    - **响应式适配**: 桌面端和移动端均能正常显示，时间轴布局自适应屏幕宽度
    - **URL 参数支持**: 预留 `loadTimelineData()` 方法支持通过 URL 参数 `?id=xxx` 加载不同时间轴数据（当前阶段使用 mock 数据兜底）

## 页面结构

> **说明**: 单页文档流布局，无导航

**页面文件**: `index.html`（或 `TimelinePage.tsx`）

| 区块 | 说明 |
|------|------|
| Header | 页面标题，从 JSON `topic` 字段读取（如"汶川地震全过程时间轴"） |
| 时间轴主体 | 竖线时间轴 + 圆点节点 + 事件卡片列表，按时间升序排列 |
| 事件卡片（收起态） | 显示时间、标题、摘要、阶段标签 |
| 事件卡片（展开态） | 额外显示详细说明、地点、来源链接列表 |

---

## 页面布局建议

- **布局模式**: 单栏居中布局，时间轴竖线位于左侧，事件卡片位于右侧（桌面端）；移动端时间轴竖线靠左，卡片占满剩余宽度
- **视觉重心**: 时间轴节点与事件卡片，通过圆点 + 竖线形成清晰的时间流向感
- **结果承载区**: 事件卡片列表即为结果承载区；初始态加载 mock 数据渲染，空数据时显示"暂无事件"占位提示

## 技术选型

- **推荐方案**: 纯 HTML/CSS/JS（单文件即可运行，无需构建工具）
- **备选方案**: React + Tailwind CSS（如用户偏好组件化开发）
- **核心逻辑封装**: `loadTimelineData()` 函数负责数据获取，`renderTimeline()` 函数负责时间轴渲染
- **响应式**: CSS Media Query 或 Tailwind 响应式断点
